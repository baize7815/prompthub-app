# Prompt MCP 通用使用指南（意图驱动版）

## 任务目标
把用户的自然语言需求路由成可执行的 MCP 流程，不绑定固定场景。
核心闭环：**理解意图 → 检索候选 → 读取模板 → 产出结果 →（可选）写入库**。

## 可用 MCP 工具
- `list_prompt_categories`：列出分类
- `get_prompt_in_category`：分类内按标题查找（exact/contains）
- `get_prompt_by_title`：按标题查找
- `search_prompts`：全库模糊搜索
- `create_prompt`：新建 prompt（需写权限）

---

## 意图分类（通用，不写死场景）

### A. 检索复用类
用户想找某类 prompt 并改造成当前任务结果。
例：摄影提词、文案提词、修图提词等。

### B. 模板执行类
用户明确点名某个 prompt，要直接套用到当前输入。
例：逆向提示词反推图片。

### C. 沉淀入库类
用户要把当前产物存成 prompt。
例：上传/新建到某分类。

### D. 探索推荐类
用户不确定用哪个 prompt，先让系统推荐候选，再执行。

---

## 通用执行流水线（固定）

### Step 1：解析目标
提取四个要素：
1. 动作：找/用/改/存
2. 主题关键词：如“摄影”“逆向”
3. 范围：指定分类或全库
4. 输出类型：分析、最终 prompt、JSON 反推、入库确认等

### Step 2：优先分类检索
1. 若用户给了分类：直接分类内查
2. 若未给分类：先 `list_prompt_categories`，做语义匹配后选择最可能分类
3. 分类内查不到：再回退全库 `search_prompts`

### Step 3：读取命中 prompt
优先级：
1. `get_prompt_in_category`（分类内精准）
2. `get_prompt_by_title`（按标题）
3. `search_prompts`（兜底）

### Step 4：执行任务
- 检索复用类：提炼命中 prompt 的结构与写法，再生成适配当前任务的新版本
- 模板执行类：按模板要求处理当前输入（如反推图片）
- 探索推荐类：先给 2-5 个候选，用户确认后执行

### Step 5（可选）：写入 Prompt
仅当用户明确“保存/上传/新建”时执行：
1. 展示待写入信息（title/category/content）
2. **必须二次确认**
3. 调用 `create_prompt`
4. 返回创建结果（title/category/id）

---

## 关键规则（硬约束）

1. **不确认不写入**：未确认 title/category/content 前，禁止调用 `create_prompt`
2. **不乱猜分类**：分类不明确时，先给候选让用户选
3. **无写权限即停止写入流程**：直接提示 owner 权限要求
4. **优先复用现有 prompt**：先查再写，避免重复内容
5. **输出可直接使用**：给用户的是可粘贴执行的结果，不是只讲思路

---

## 输出模板（建议）

### 模板 1：检索复用类输出
1. 命中结果摘要（标题 + 分类）
2. 写法分析（结构、语气、约束、负向约束）
3. 生成后的最终 prompt（可直接复制）

### 模板 2：模板执行类输出
1. 使用的模板标题
2. 执行结果（如反推 JSON / 结构化说明）
3. 可选：再给一个“更保守/更激进”版本

### 模板 3：入库类输出
1. 待写入预览（title/category/content）
2. 用户确认记录
3. 创建结果（ID + 标题 + 分类）

---

## 异常处理

- 分类不存在：返回可用分类列表并让用户选择
- 分类内无匹配：自动全库搜索兜底
- 全库仍无匹配：让用户补充关键词或直接提供原始模板
- 标题重复：提示改名后重试
- 写入失败：保留 content，提示用户稍后重试写入

---

## 触发语句参考（示例，不限于此）

- “在生图里找摄影类提词，分析后帮我写一个适配这张图的”
- “用逆向提示词模板反推这张图”
- “把上面的提示词保存成 prompt 到生图分类”
- “我不确定该用哪个提示词，你先给我推荐几个”
